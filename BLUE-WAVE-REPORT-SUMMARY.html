📁 What Was Built

A fully automated, progressive news aggregator website — styled like Drudge Report but with a blue/navy design, Oswald font, and left-leaning sources. The site regenerates itself every day automatically via GitHub Actions.

🚀 How to Publish (Step by Step)

Step 1 — Install Git (one time only)

Download from git-scm.com/download/win and install with default settings.

Step 2 — Create a GitHub account

Go to github.com → Sign up (free). Enable two-factor authentication in Settings → Security.

Step 3 — Create a new repository

Click the + icon → New repository → Name it blue-wave-report → Public → Create.

Step 4 — Push your files

Open Command Prompt, navigate to your folder, and run:

# Navigate to your project folder
cd "C:\Users\sasha\OneDrive\Desktop\Claude\blue-wave-report"

# Initialize git and push
git init
git add .
git commit -m "Launch BLUE WAVE BEACON"
git remote add origin https://github.com/YOUR_USERNAME/blue-wave-report.git
git branch -M main
git push -u origin main

Step 5 — Enable GitHub Pages

In your GitHub repo → Settings → Pages → Source: Deploy from branch → Branch: main → Folder: / (root) → Save.

Your site will be live at: https://YOUR_USERNAME.github.io/blue-wave-report

Step 6 — Add a custom domain (optional, ~$10/year)

1. Buy a domain at namecheap.com or cloudflare.com
2. Sign up free at cloudflare.com, add your domain
3. In GitHub Pages Settings → Custom domain → enter your domain
4. Cloudflare gives you free SSL, CDN, and DDoS protection automatically

Step 7 — Verify auto-updates are working

Go to your repo → Actions tab. You should see "Daily News Update" scheduled. You can click Run workflow to trigger it manually and test it immediately.

⚙️ Auto-update schedule: The site regenerates every day at 7:00 AM Eastern automatically. GitHub fetches all RSS feeds, pulls Bluesky trending posts, rebuilds index.html, and commits the changes — no action needed from you.

🔒 Security Overview

  • No database — can't be SQL injected
  • No login system — no credentials to steal
  • No server-side code in production
  • All links use noopener noreferrer
  • HTML-escaped RSS content (no injection)
  • GitHub Actions sandboxed — isolated
  • Donations go directly to payment platforms
  • You never touch money or card data

⚠️ Things to Do

  • Enable GitHub 2FA (two-factor auth)
  • Add security headers in Cloudflare
  • Add honeypot field to newsletter form
  • Use a strong unique GitHub password
  • Don't commit any API keys to the repo

Add Security Headers (Cloudflare — Free)

In Cloudflare → Rules → Transform Rules → Modify Response Header, add:

X-Frame-Options: DENY
X-Content-Type-Options: nosniff
Referrer-Policy: strict-origin-when-cross-origin
Permissions-Policy: geolocation=(), microphone=(), camera=()

Newsletter Spam Protection

Add this hidden field inside the newsletter <form> tag in index.html to block bots:

<input type="text" name="_gotcha" style="display:none" />

🕵️ Staying Anonymous (Privacy Setup)

Goal: keep your real name and address off the public internet while still following the rules (ICANN legally requires registrars to collect accurate info — you can't use fake details or the domain can be cancelled).

Do This When You Register

  • Register the domain through Cloudflare — it includes free WHOIS privacy automatically, so the public WHOIS record shows Cloudflare's proxy info instead of your name/address.
  • Use your ProtonMail address for the Cloudflare account, the domain registration, GitHub, and Formspree — not the personal Hotmail (buff_ss@hotmail.com). This keeps your real-name email off everything.
  • Confirm "WHOIS / Domain Privacy" is ON before finishing checkout (Cloudflare defaults to on, but double-check).
  • Site contact email is already set to contact@bluewavebeacon.com — set that up as a forwarding alias to your ProtonMail, so you never expose a personal address on the site.
  • GitHub: pick a username that isn't your real name, and in Settings → Emails enable "Keep my email address private" so commits don't leak your email.

What This Does & Doesn't Hide

WhoCan see your real identity?
Public / site visitors / WHOIS lookups❌ No — hidden by WHOIS privacy
Cloudflare (the company)✅ Yes — legally required to collect it
Law enforcement (with a subpoena/court order)✅ Yes — can be compelled

Bottom line: Cloudflare's free WHOIS privacy + a dedicated ProtonMail address = anonymous to the public, which is plenty for a news aggregator. If you ever need stronger anonymity (e.g. whistleblower-level), a privacy registrar like Njalla registers the domain in their own name on your behalf — but you then trust Njalla instead of Cloudflare.
